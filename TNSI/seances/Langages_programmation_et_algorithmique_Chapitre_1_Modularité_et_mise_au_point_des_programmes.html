📘 Chapitre 1 : Modularité et mise au point des programmes (Python)

🎯 Objectifs du chapitre

  • Comprendre la notion de modularité d’un programme
  • Structurer son code avec des fonctions ou fichiers
  • Déboguer un programme Python

🔹 Qu'est-ce que la modularité ?

La modularité permet de diviser un programme en plusieurs fonctions ou modules. Cela facilite :

  • la lecture du code
  • la maintenance
  • la réutilisation

📌 Exemple en Python :

def calcul_moyenne(notes):
    return sum(notes) / len(notes)

mes_notes = [14, 16, 13]
print("Moyenne :", calcul_moyenne(mes_notes))

🔎 Mise au point (debug)

Pour corriger ou analyser le comportement d'un programme :

  • print() : pour afficher une variable
  • type() : pour connaître le type d'une variable

📌 Exemple :

eleve = {"nom": "Lina", "age": 17}
print(eleve)
print("Nom :", eleve["nom"])
print("Type :", type(eleve))

📁 Organisation du code en modules

On peut séparer le code dans plusieurs fichiers Python et les importer :

📂 Exemple :

fichier : outils.py

def bonjour(prenom):
    return f"Bonjour {prenom} !"

fichier : main.py

from outils import bonjour

print(bonjour("Thomas"))

✅ Bonnes pratiques

  • Donner des noms explicites aux fonctions et variables
  • Tester les fonctions séparément
  • Ajouter des commentaires si nécessaire
