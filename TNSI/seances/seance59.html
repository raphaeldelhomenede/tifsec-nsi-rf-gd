Séance 59 : Schéma Relationnel

1. Qu'est-ce qu'un schéma relationnel ?

Un schéma relationnel est une représentation formelle des données d'une base de données relationnelle.

2. Définition des entités et des relations

Dans une base de données, on modélise le monde réel sous forme de tables, appelées aussi entités. Les relations entre ces entités sont décrites par des relations.

Exemple :

Dans une école, nous avons des élèves et des cours :

  • L'entité Élève a des attributs : id, nom, prénom, classe.
  • L'entité Cours a des attributs : id, nom, professeur.
  • Un élève peut suivre plusieurs cours, et un cours peut être suivi par plusieurs élèves.

Nous avons donc une relation "suivre" entre Élève et Cours.

3. Notation et types de relations

Les relations sont exprimées par des phrases avec des verbes à l’infinitif.

Exemple : "Un élève s'inscrire à un cours".

On distingue plusieurs types de relations :

  • 1-1 (un à un) : Exemple : Un passeport appartient à un seul individu.
  • 1-n (un à plusieurs) : Exemple : Un professeur enseigne plusieurs cours.
  • n-n (plusieurs à plusieurs) : Exemple : Un élève peut suivre plusieurs cours et un cours peut être suivi par plusieurs élèves.

Notation sous forme relationnelle :

        ÉLÈVE (id_élève, nom, prénom, classe)
        COURS (id_cours, nom, professeur)
        INSCRIPTION (id_élève, id_cours)

Comment visualiser les relations ?

Pour mieux comprendre les relations, utilisez des schémas entité-association avec des outils comme DB Designer ou draw.io.

Vous pouvez aussi passer d'une base de données à un graphique via les sites suivant :

  • Lucidchart : Cet outil de modélisation de base de données gratuit prend en charge des plateformes SGBD telles que MySQL, Oracle, PostgreSQL et Microsoft SQL Server. Vous pouvez importer la structure de votre base de données directement dans Lucidchart pour la visualiser sous forme de diagramme entité-association. LUCIDCHART.COM
  • DbSchema : DbSchema simplifie la conception et la gestion des bases de données, vous permettant de créer, visualiser et maintenir facilement vos structures de base de données. Il prend en charge l'ingénierie inverse du schéma à partir de bases de données locales, distantes ou dans le cloud. DBSCHEMA.COM
  • dbdiagram.io : Un outil gratuit et simple qui vous aide à dessiner rapidement vos diagrammes de relations de base de données en utilisant un langage DSL simple. DBDIAGRAM.IO
  • Creately : Un outil de conception de base de données en ligne qui permet de visualiser des systèmes de bases de données complexes avec un minimum d'effort. Il offre des bibliothèques de formes standard pour les bases de données et des centaines de modèles de diagrammes pré-dessinés. CREATELY.COM
  • ONDA (ONline Database Architect) : Un outil web développé à l'Université de Coimbra qui permet la création de diagrammes entité-association, la visualisation de modèles physiques et la génération de code SQL pour divers moteurs de base de données. ARXIV.ORG

4. Clés primaires et clés étrangères

Les relations entre les tables sont gérées grâce aux clés primaires et clés étrangères.

  • Clé primaire : Identifie de manière unique un enregistrement dans une table.
  • Clé étrangère : Fait référence à une clé primaire d'une autre table.
        CREATE TABLE Eleve (
            id_élève INT PRIMARY KEY,
            nom VARCHAR(50),
            prénom VARCHAR(50),
            classe VARCHAR(10)
        );
        
        CREATE TABLE Cours (
            id_cours INT PRIMARY KEY,
            nom VARCHAR(50),
            professeur VARCHAR(50)
        );
        
        CREATE TABLE Inscription (
            id_élève INT,
            id_cours INT,
            PRIMARY KEY (id_élève, id_cours),
            FOREIGN KEY (id_élève) REFERENCES Eleve(id_élève),
            FOREIGN KEY (id_cours) REFERENCES Cours(id_cours)
        );

5. Exercices ludiques

Exercice 1 : Identifier les relations

Une entreprise a des employés qui travaillent dans des départements. Chaque employé peut être dans un seul département, mais un département peut avoir plusieurs employés.

Question : Identifiez les entités, leurs attributs et la relation entre elles.

Exercice 2 : Compléter la notation relationnelle

Une université gère des étudiants, des matières et des enseignants. Chaque matière est enseignée par un enseignant et suivie par plusieurs étudiants.

À vous de jouer !

Exercice 3 : Construire un schéma relationnel

Modélisez un système de location de voitures, en définissant les entités, les attributs et les relations.

Conclusion

Le schéma relationnel permet de structurer une base de données et d'éviter les incohérences. En comprenant bien les relations et les clés, on peut modéliser efficacement n'importe quel système.
