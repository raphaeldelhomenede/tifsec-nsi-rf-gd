Bases de Données
Chapitre 2 : Systèmes de Gestion de Bases de Données - SQL
1. Qu'est-ce qu'un SGBD ?
Un Système de Gestion de Base de Données (SGBD) est un logiciel qui permet de créer, manipuler, administrer et interroger une base de données. Il assure la cohérence, la sécurité et l'efficacité de l'accès aux données.
  • Exemples : MySQL, PostgreSQL, Oracle, SQL Server, SQLite
  • Langage standard utilisé : SQL (Structured Query Language)

2. Architecture d'un SGBD
  • Niveau logique : tables, vues, contraintes
  • Niveau physique : fichiers sur disque, index
  • Moteur SQL : analyse et exécution des requêtes
Le SGBD masque les détails physiques aux utilisateurs pour leur fournir une vue logique cohérente et simplifiée.
3. SQL : Définition et catégories
  • DDL (Data Definition Language) : création et modification de structures (CREATE, ALTER, DROP)
  • DML (Data Manipulation Language) : manipulation des données (SELECT, INSERT, UPDATE, DELETE)
  • DCL (Data Control Language) : contrôle d'accès (GRANT, REVOKE)
4. DDL - Création de tables
CREATE TABLE Client ( id INT PRIMARY KEY, nom VARCHAR(100), email VARCHAR(100) );
CREATE TABLE Commande ( id INT PRIMARY KEY, date_commande DATE, id_client INT, FOREIGN KEY (id_client) REFERENCES Client(id) );
5. DML - Manipulation des données
-- Insertion de données INSERT INTO Client VALUES (1, 'Alice', 'alice@mail.com'); -- Mise à jour UPDATE Client SET email = 'alice2024@mail.com' WHERE id = 1; -- Suppression DELETE FROM Client WHERE id = 1; -- Requête de sélection SELECT * FROM Client; SELECT nom FROM Client WHERE email LIKE '%@mail.com';
6. Jointures
Les jointures permettent de combiner des données issues de plusieurs tables.
  • INNER JOIN : correspondance entre deux tables
  • LEFT JOIN : toutes les lignes de gauche, même sans correspondance
SELECT Client.nom, Commande.date_commande FROM Client INNER JOIN Commande ON Client.id = Commande.id_client;
7. Vues (VIEW)
Une vue est une table virtuelle basée sur une requête SQL. Elle permet de simplifier les accès ou de restreindre les données visibles.
CREATE VIEW ClientsCommandes AS SELECT Client.nom, Commande.date_commande FROM Client INNER JOIN Commande ON Client.id = Commande.id_client;
8. Transactions
Une transaction est une série d'opérations SQL exécutées comme une unité indivisible. Elle respecte les propriétés ACID :
  • Atomicité : tout ou rien
  • Constance : respecte les règles d'intégrité
  • Isolation : pas d'interférence entre transactions
  • Durabilité : les données sont persistées
BEGIN TRANSACTION; UPDATE Compte SET solde = solde - 100 WHERE id = 1; UPDATE Compte SET solde = solde + 100 WHERE id = 2; COMMIT;
9. Résumé
  • Un SGBD permet de gérer efficacement une base de données
  • Le langage SQL permet de définir, manipuler et contrôler les données
  • Les transactions assurent l'intégrité des données
  • Les vues et jointures permettent des accès personnalisés et performants
