Structures de Données

Chapitre 1 : Interface et implémentation des structures de données

Introduction générale

Pour créer des programmes efficaces, il faut savoir comment organiser les données. C’est le rôle des structures de données. Elles permettent de stocker, retrouver, et manipuler les données de manière rapide et organisée.

Dans ce chapitre, on va apprendre deux idées clés :

  • L’interface : ce qu’on peut faire avec la structure
  • L’implémentation : comment cela fonctionne à l’intérieur

1. Qu’est-ce qu’une structure de données ?

Une structure de données est un modèle pour organiser des données. Elle permet de gérer plusieurs valeurs ensemble. Chaque structure est adaptée à un besoin précis.

Exemple 1 : Liste (comme une liste de courses)

courses = ["lait", "pain", "œufs"]
courses.append("bananes")  # Ajoute un élément
courses[1]  # Donne "pain"

Exemple 2 : Pile (LIFO - Last In First Out)

pile = []
pile.append("dossier1")  # On empile
pile.append("dossier2")
pile.pop()  # On retire le dernier ajouté → "dossier2"

Exemple 3 : File (FIFO - First In First Out)

from collections import deque
file = deque()
file.append("client1")
file.append("client2")
file.popleft()  # client1 sort en premier

2. L’interface d’une structure

L’interface, c’est la liste des actions disponibles. Elle décrit ce qu’on peut faire avec la structure, mais pas comment cela fonctionne.

Exemple : Interface d’une pile

  • empiler(x) → ajouter un élément
  • dépiler() → retirer le dernier
  • sommet() → voir le dernier sans le retirer
  • estVide() → vérifier si la pile est vide

Exemple : Interface d’une file

  • enfiler(x) → ajouter en fin
  • défiler() → retirer en début
  • tête() → voir le premier

Une interface est comme un contrat. Tant que la structure respecte ce contrat, l’utilisateur peut l’utiliser sans connaître son fonctionnement interne.

3. L’implémentation d’une structure

L’implémentation est la partie cachée : elle décrit comment l’interface est réalisée techniquement.

Implémenter une pile avec un tableau :

pile = []

def empiler(x):
    pile.append(x)

def dépiler():
    return pile.pop()

def sommet():
    return pile[-1]

def estVide():
    return len(pile) == 0

Implémenter une pile avec une liste chaînée :

class Noeud:
    def __init__(self, valeur, suivant=None):
        self.valeur = valeur
        self.suivant = suivant

class Pile:
    def __init__(self):
        self.tete = None

    def empiler(self, val):
        self.tete = Noeud(val, self.tete)

    def dépiler(self):
        val = self.tete.valeur
        self.tete = self.tete.suivant
        return val

    def estVide(self):
        return self.tete is None

Les deux implémentations font exactement la même chose, mais le "moteur" est différent.

4. Analogie simple

Une voiture : tu tournes le volant (interface), mais tu ne sais pas exactement ce que fait le moteur ou l’ordinateur de bord (implémentation). Tu te fiches de savoir comment ça marche tant que ça marche.

C’est pareil avec les structures de données : tu veux juste que ça fasse ce que tu as demandé.

6. Résumé du chapitre

  • Structure de données : outil pour stocker et organiser les données
  • Interface : liste des opérations disponibles
  • Implémentation : code réel qui fait fonctionner l’interface
  • On peut changer l’implémentation sans changer l’interface
