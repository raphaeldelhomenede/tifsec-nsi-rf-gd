📘 Chapitre 1 : Modularité et mise au point des programmes (Python)

🎯 Objectifs pédagogiques

  • Comprendre la notion de modularité dans la programmation
  • Savoir organiser un programme avec des fonctions et des modules
  • Développer une méthode de test et de débogage
  • Utiliser des outils de mise au point comme les assertions et les exceptions

🔹 Qu'est-ce que la modularité ?

La modularité est une manière de concevoir un programme informatique en le décomposant en parties indépendantes, appelées modules ou fonctions.

Chaque module/fonction remplit une tâche précise. Cela permet de :

  • Faciliter la compréhension du code
  • Réutiliser des blocs de code
  • Tester les composants individuellement
  • Améliorer la maintenance du programme

📌 Exemple simple :

def addition(a, b):
    return a + b

def soustraction(a, b):
    return a - b

def main():
    x = 5
    y = 3
    print("Addition:", addition(x, y))
    print("Soustraction:", soustraction(x, y))

main()

Le programme est composé de plusieurs fonctions indépendantes. Elles peuvent être testées séparément.

📁 Organisation du code : fichiers et modules

Quand un programme devient complexe, il est préférable de séparer les fonctions dans des fichiers appelés modules.

🧩 Exemple :

util.py

def est_pair(n):
    return n % 2 == 0

main.py

from util import est_pair

n = int(input("Entrez un nombre : "))
if est_pair(n):
    print("Ce nombre est pair.")
else:
    print("Ce nombre est impair.")

Le fichier util.py est un module contenant une fonction. Il est importé dans main.py.

🛠️ Mise au point (debugging)

La mise au point permet de détecter et corriger les erreurs d’un programme. Les outils classiques sont :

  • print() pour afficher la valeur d’une variable
  • assert pour vérifier une hypothèse
  • try/except pour capturer les erreurs et éviter les plantages

🧪 Exemple : print pour déboguer

def divise(a, b):
    print("a:", a, "b:", b)
    return a / b

print(divise(10, 2))

⚠️ Exemple avec exception :

def division(a, b):
    try:
        return a / b
    except ZeroDivisionError:
        return "Erreur : division par zéro."

print(division(5, 0))

✅ Exemple avec assert :

def carre(x):
    return x * x

assert carre(3) == 9
assert carre(0) == 0
assert carre(-2) == 4

🔬 Importance des tests

Tester un programme signifie vérifier qu'il produit les bons résultats dans différents cas :

  • Cas standards
  • Cas limites
  • Cas d’erreurs

On peut écrire des tests unitaires pour automatiser cette vérification.

🧪 Exemple de test unitaire :

def maximum(a, b):
    return a if a > b else b

def test_maximum():
    assert maximum(5, 3) == 5
    assert maximum(0, 0) == 0
    assert maximum(-2, 1) == 1

# Exécution des tests

print("Lancement des tests...")
test_maximum()
print("Tous les tests sont passés !")

📌 Documentation et commentaires

Commenter son code permet de mieux le comprendre et de le rendre plus lisible par les autres développeurs (ou soi-même plus tard !).

Docstring (documentation intégrée) :

def somme(a, b):
    """Retourne la somme de deux nombres."""
    return a + b

🧰 Bonnes pratiques

  • Découper le programme en fonctions de petite taille
  • Donner des noms clairs aux fonctions et variables
  • Documenter avec des commentaires ou docstrings
  • Tester chaque fonction séparément
  • Préférer l'importation de modules plutôt que copier/coller du code

🧩 Exemple complet

# fichier util.py

def est_multiple(a, b):
    """Retourne True si a est multiple de b."""
    return a % b == 0

# fichier main.py
from util import est_multiple

n = int(input("Entrez un nombre : "))
m = int(input("Diviseur ? : "))

if est_multiple(n, m):
    print(f"{n} est un multiple de {m}")
else:
    print(f"{n} n'est PAS un multiple de {m}")
