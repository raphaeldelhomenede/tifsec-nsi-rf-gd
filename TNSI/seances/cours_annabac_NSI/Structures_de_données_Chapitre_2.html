Structures de Données

Chapitre 2 : Structures Arborescentes

1. Introduction aux arbres

Une structure arborescente est une manière d'organiser les données en forme d'arbre. C'est une structure hiérarchique composée de nœuds.

  • Un nœud racine (le sommet de l'arbre)
  • Des nœuds enfants
  • Des nœuds feuilles (sans enfants)

2. Vocabulaire de base

  • Racine : le premier nœud de l'arbre
  • Parent / Enfant : relation entre deux nœuds connectés
  • Feuille : un nœud sans enfants
  • Hauteur : nombre de niveaux de l'arbre

3. Exemple simple d'un arbre

        A  ← racine
       / \
      B   C ← enfants de A
     / \
    D   E ← enfants de B

Dans cet exemple, A est la racine, B et C sont les enfants de A, et D, E sont les feuilles.

4. Parcours d'un arbre

  • Préfixe (pré-ordre) : on visite le nœud, puis les enfants
  • Infixe (in-ordre) : on visite l'enfant gauche, le nœud, puis l'enfant droit
  • Postfixe (post-ordre) : on visite les enfants, puis le nœud

Exemple : Parcours préfixe

        A
       / \
      B   C
     / \
    D   E

      → Préfixe : A, B, D, E, C

5. Implémentation simple en Python

class Noeud:
    def __init__(self, valeur):
        self.valeur = valeur
        self.enfants = []

    def ajouter_enfant(self, enfant):
        self.enfants.append(enfant)

racine = Noeud("A")
b = Noeud("B")
c = Noeud("C")
d = Noeud("D")
e = Noeud("E")

racine.ajouter_enfant(b)
racine.ajouter_enfant(c)
b.ajouter_enfant(d)
b.ajouter_enfant(e)

6. Utilisations des arbres

  • Organiser des fichiers (explorateur de fichiers)
  • Créer des expressions mathématiques
  • Représenter des hiérarchies (entreprise, taxonomie)
  • Moteurs de recherche (arbre de décision, IA)

7. Types d'arbres

  • Arbre binaire : chaque nœud a au plus 2 enfants
  • Arbre binaire de recherche (BST) : pour trier les données
  • Arbre N-aire : chaque nœud peut avoir N enfants
  • Trie : pour les dictionnaires et l'autocomplétion

8. Résumé du chapitre

  • Les arbres modélisent des structures hiérarchiques
  • Un arbre est composé de nœuds liés entre eux
  • Il existe plusieurs parcours possibles : préfixe, infixe, postfixe
  • De nombreuses applications concrètes : fichiers, IA, etc.
