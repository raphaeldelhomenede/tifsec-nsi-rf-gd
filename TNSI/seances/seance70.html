Séance 70 : Les Mémoires et Stockage - Escape Game Numérique

Objectif :

Comprendre les différents types de mémoire informatique (RAM, ROM, Flash, SSD, HDD, etc.) à travers une approche ludique et interactive. La séance est divisée en deux parties : une première partie sous forme d’escape game où les élèves doivent résoudre des énigmes liées aux mémoires, et une seconde partie où ils manipuleront un RAM Disk pour comparer les performances des différentes technologies de stockage.

1. Introduction aux Mémoires Informatiques

Avant de plonger dans l’escape game, il est essentiel de comprendre les bases des différents types de mémoire. Voici un récapitulatif des principales catégories :

Type de Mémoire Caractéristiques Exemple d’Utilisation
RAM (Random Access Memory) Volatile, rapide, utilisée pour stocker temporairement les données actives. Mémoire vive des ordinateurs, des consoles de jeux.
ROM (Read Only Memory) Non-volatile, ne peut pas être modifiée, contient le BIOS. Firmware des appareils électroniques (routeurs, microcontrôleurs).
Flash Non-volatile, plus rapide que les disques mécaniques, utilisé dans les SSD et clés USB. Stockage des systèmes d’exploitation, téléphones.
HDD (Disque Dur Mécanique) Stockage magnétique, capacité élevée mais plus lent que les SSD. Stockage de masse (serveurs, anciens PC).
SSD (Solid State Drive) Utilise la mémoire flash, beaucoup plus rapide qu’un HDD. Stockage principal des PC modernes.

2. Escape Game - "Mission Récupération de Données"

📜 Scénario :

Un virus informatique a corrompu les données essentielles d’un serveur. Votre mission est de restaurer les fichiers en résolvant trois énigmes. Chaque défi réussi vous fournira un indice crucial pour l’énigme finale.

⚠️ Attention : Le temps est compté ! Vous devez récupérer les données avant que le serveur ne s’éteigne définitivement.

🎯 Déroulement de l’Escape Game :

  • Objectif : Comprendre la mémoire en informatique (RAM, SSD, HDD, latence).
  • Organisation : Travail en équipe.
  • Récompenses : Chaque défi réussi débloque un indice pour résoudre l’énigme finale.

🏆 Défi 1 : Comment fonctionne la RAM ?

Objectif :

Comprendre pourquoi la RAM est essentielle au fonctionnement d’un ordinateur.

📝 Activité :

  1. Les élèves reçoivent une fiche expliquant le rôle de la RAM et un tableau à compléter.
  2. Ils doivent classer les éléments suivants en Mémoire vive (RAM) ou Mémoire de stockage (HDD/SSD) :
Élément RAM ou Stockage ?
Exécution d’un programme ❓
Fichier enregistré sur un disque ❓
Onglet ouvert dans un navigateur ❓
Photo stockée sur un téléphone ❓

🏆 Défi 2 : Pourquoi les SSD sont plus rapides que les HDD ?

Objectif :

Comprendre la différence de fonctionnement entre les HDD et SSD.

🔍 Activité :

  1. Les élèves observent un schéma comparant un disque dur mécanique (HDD) et un SSD.
  2. Ils doivent répondre à un quiz rapide en équipe :
  • 💡 Pourquoi un SSD est-il plus rapide qu’un HDD ?
  • 💡 Quel élément mécanique rend le HDD plus lent ?
  • 💡 Quelle mémoire consomme le moins d’énergie ?

🏆 Défi 3 : Qu’est-ce que la latence mémoire ?

Objectif :

Comprendre pourquoi certaines mémoires sont plus lentes que d’autres.

🔬 Expérience interactive :

  1. Un chronomètre est affiché à l’écran.
  2. Les élèves doivent simuler une file d’attente :
    • 💾 Le premier élève est un HDD (il doit tourner trois fois sur lui-même avant de répondre).
    • ⚡ Un autre élève est un SSD (il répond immédiatement).
    • 🧠 Un dernier élève est la RAM (il répond encore plus vite).
  3. Comparaison des temps de réponse et discussion sur la latence.

🔎 Énigme Finale : Restaurer le fichier corrompu

🔧 Mission :

Les élèves doivent utiliser les trois indices obtenus pour résoudre l’énigme :

"Quelle technologie utiliser pour restaurer les données du serveur le plus rapidement possible ?"

📌 Réponse attendue :

La RAM est la plus rapide mais ne conserve pas les données.
Le SSD est rapide et stable.
Le HDD est trop lent pour une récupération immédiate.

Solution : Utiliser la **RAM pour stocker temporairement les données**, puis les enregistrer sur un **SSD**.

🎉 Conclusion et Discussion

  • Quels sont les points forts/faibles de chaque type de mémoire ?
  • Pourquoi ne pas utiliser uniquement la RAM si elle est si rapide ?
  • Quels sont les avantages des SSD sur les HDD ?

À travers cette mission, les élèves ont découvert la mémoire informatique de manière ludique et ont expérimenté l’impact de la latence et du type de stockage sur les performances d’un système.

✨ Félicitations, agents ! Mission accomplie ! 🚀 ✨


1. Création d’un RAM Disk

💡 Un RAM Disk est un espace de stockage temporaire utilisant la mémoire vive. Il est beaucoup plus rapide qu’un disque dur ou un SSD mais disparaît à l’extinction de l’ordinateur.

Manipulation en ligne de commande :


# Création d’un RAM Disk de 512 Mo sous Linux
sudo mount -t tmpfs -o size=512M tmpfs /mnt/ramdisk

# Vérification de la création
df -h /mnt/ramdisk

# Copie d’un fichier pour test
cp fichier_test.txt /mnt/ramdisk/

2. Comparaison SSD vs RAM Disk

📊 Les élèves doivent **tester la vitesse d’écriture** en copiant un fichier sur un SSD et sur le RAM Disk.

Commande pour mesurer le temps d’écriture :


# Écriture d’un fichier sur SSD
time cp fichier_test.txt /home/utilisateur/

# Écriture sur RAM Disk
time cp fichier_test.txt /mnt/ramdisk/

📌 Discussion sur les résultats :

  • Pourquoi le RAM Disk est plus rapide ?
  • Quels sont ses inconvénients (perte des données à l’extinction) ?

  • Comment créer un disque temporaire en RAM.
